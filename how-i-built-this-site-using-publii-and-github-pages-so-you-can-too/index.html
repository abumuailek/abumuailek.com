🚀 How I Built This Site Using Publii & GitHub Pages (So You Can Too)

Hello there, internet wanderer! 👋

Welcome to the humble digital corner I now call abumuailek.com — a place where I’ll post my work, ideas, code experiments, occasional caffeine-induced rants, and maybe pictures of cats (don’t judge).

I built this site with a simple goal: have a personal space that’s lightweight, fast, good-looking on your phone, and stupid-easy to update. Oh, and it had to be free. I'm a techie, not a magician 🧙‍♂️.

So here's how I did it — and how you can do the same in a couple of hours (or 7, if you’re like me and tweak fonts for 3 of them).

💡 The Idea

I wanted:

  • Something I control (no Medium, Substack, etc.)

  • A CMS-like interface, but without a WordPress backend or paying for hosting

  • Easy syncing with GitHub Pages

  • Minimalism, but not boring

After some research (read: procrastinating on YouTube), I found a tool called Publii. It’s a free desktop app that lets you create static websites with CMS-like ease. No PHP, no MySQL, no server drama. Just click → write → sync to GitHub. Boom.

🛠️ What I Used

  • 🖥️ Publii – the core of the project

  • 🧑‍💻 GitHub Pages – the free hosting

  • 🌐 Cloudflare – I already had my domain here

  • 🎨 A clean Publii theme with just enough personality

  • ☕ My over-caffeinated attention span

🧭 Step-by-Step Guide: Build Yours

Step 1: Install Publii

  • Download Publii for Windows, macOS, or Linux

  • Install. Launch. Take a moment to appreciate how sleek it looks.

Step 2: Create a New Website

  • Click “Create New Website”

  • Give it a name (I used “abumuailek” because, well, that’s me).

  • Choose a theme. I went minimal. You do you.

Step 3: Add Your Content

  • Use Publii’s editor to write pages and blog posts.

  • It supports markdown, images, tags, SEO settings — the works.

  • I created:

    • A homepage

    • An “About Me”

    • A link to my resume (PDF uploaded to static files)

    • Blog section (like this post!)

Step 4: Connect to GitHub Pages

  • Go to “Server” settings → Choose GitHub Pages.

  • Enter:

    • Your GitHub username

    • Repo name: yourusername.github.io

    • Branch: main (or master, if you're retro)

    • Optional: Custom domain like abumuailek.com

  • Hit “Test Connection” to make sure it works.

  • Click “Sync your website” and voila!

Step 5: Point Your Domain

  • I use Cloudflare, but most registrars work.

  • Add a CNAME record:

    Name: @
Target: yourusername.github.io
TTL: Auto

  • GitHub will auto-serve the site after a few minutes.

🧩 Bonus Touches

  • Embedded my GitHub contributions

  • Added SEO titles and meta descriptions via Publii

  • Tweaked some theme settings to feel more “me”

  • Will likely keep obsessively changing the favicon until it’s perfect

😅 Lessons Learned

  • Publii is a game-changer for people who want to own their space but still want the ease of writing like it’s Medium.

  • GitHub Pages is reliable, fast, and free — what's not to love?

  • Don’t overthink design… at first. Get content live, tweak later.

  • Backup before syncing. Learned this the “oops-my-homepage-is-gone” way.

🧠 Why You Should Try This

If you're:

  • A dev or tinkerer wanting a personal home

  • A writer who hates WordPress bloat

  • Someone tired of renting space on platforms

Then this combo — Publii + GitHub Pages — is your minimalist power move.
